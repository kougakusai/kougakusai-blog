またこの記事はsample.mdx というファイルですがファイルの先頭には以下のような記述がされています。 
---
title: 記事の書き方(ここがタイトル)
description: ブログ記事の書き方を教えたいんじゃ！(記事の見出し)
updatedAt: 2021/01/20(記事作成日)
---

これは FrontMatter という記法です。

content/posts に.mdxファイルを作りこのように記述すれば自動的に記事一覧ページに追加されます。

つまりファイルを作って FrontMatter で書きさえすればページが作れる。後は夢中になって内容書くだけ。

次に、技術ブログではコードを紹介したいことがあるかと思います。

シンタックスハイライトを効かせましょう。

```拡張子

//表示したいコード

```

このように表示したいコードを逆クオート 3 つで囲むことで表示できます。今回は拡張子 go を指定してみます。

package main
import "fmt"
func main() {
    fmt.Println("Hello, World!")
}

※動画の内容はまったく関係ありません。僕が 1 番最近いいねした動画です。

MDXはMarkdownにJSXを挿入できるフォーマットです。つまりMarkdownで書くブログ記事にimportしたコンポーネントを組み込むことができます。多くの参考サイトではファイル毎にコンポーネントをimportしていましたがいちいちimportするのもめんどくさく感じたので component/layout.js 内のMDXProviderに使いたいコンポーネントを一気に渡しました。そうすることでimportしなくてもすべての.mdxファイル内でそれらのコンポーネントを使えるようにしました。この判断がブログの運営において吉と出るか凶と出るかは定かではないので、さらなる改善改良を期待したいと思います。💩
